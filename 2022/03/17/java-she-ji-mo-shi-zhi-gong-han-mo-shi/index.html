设计模式学习笔记：bugstack 虫洞栈

开发环境

  • JDK 1.8
  • IDEA
  • Maven

工厂模式介绍

工厂模式又称工厂方法模式，是一种创建型设计模式，其在父类中提供一个创建对象的方法，允许子类决定实例化对象的类型。

优点：提供代码结构的扩展性，屏蔽每一个功能类中的具体实现逻辑，让外部可以更加简单的只是知道调用即可。

缺点：需要实现的类非常多。

实战

模拟积分兑换中的发放多种类型奖品。

代码实现

定义奖品类型

public enum CommodityType {
    /** 兑换卷 */
    Card,
    /** 优惠卷 */
    Coupon,
    /** 商品 */
    Goods;
}

创建全局的奖品定义，有利于今后对代码的维护工作。

定义发奖接口

public interface ICommodity {
    void sendCommodity(String uId, String commodityId, String bizId, Map<String, String> extMap) throws Exception;
}

实现接口

兑换卷
@Slf4j
public class CardCommodityService implements ICommodity {

    @Override
    public void sendCommodity(String uId, String commodityId, String bizId, Map<String, String> extMap) throws Exception {
        log.info("请求参数[兑换券] => uId：{} commodityId：{} bizId：{} extMap：{}", uId, commodityId, bizId, JSON.toJSON(extMap));
    }
}
优惠卷
@Slf4j
public class CouponCommodityService implements ICommodity {

    @Override
    public void sendCommodity(String uId, String commodityId, String bizId, Map<String, String> extMap) throws Exception {
        log.debug("请求参数[优惠券] => uId：{} commodityId：{} bizId：{} extMap：{}", uId, commodityId, bizId, JSON.toJSON(extMap));
    }
}
奖品
@Slf4j
public class GoodsCommodityService implements ICommodity {

    @Override
    public void sendCommodity(String uId, String commodityId, String bizId, Map<String, String> extMap) throws Exception {
        log.info("请求参数[商品] => uId：{} commodityId：{} bizId：{} extMap：{}", uId, commodityId, bizId, JSON.toJSON(extMap));
    }
}

定义奖品工程类

public class StoreFactory {

    public ICommodity getCommodityService(CommodityType commodityType) {
        if (commodityType.equals(CommodityType.Coupon)) { return new CouponCommodityService(); }
        if (commodityType.equals(CommodityType.Goods)) { return new GoodsCommodityService(); }
        if (commodityType.equals(CommodityType.Card)) { return new CardCommodityService(); }
        return null;
    }
 }

使用

调用统一的 public ICommodity getCommodityService(CommodityType commodityType) 方法，根据不同奖品类型返回不同的实现类，达到发放不同奖品的目的。

public static void main(String[] args) throws Exception {

    StoreFactory storeFactory = new StoreFactory();
    // 1. 优惠券
    ICommodity commodityService_1 = storeFactory.getCommodityService(CommodityType.Coupon);
    commodityService_1.sendCommodity("10001", "EGM1023938910232121323432", "791098764902132", null);
    // 2. 实物商品
    ICommodity commodityService_2 = storeFactory.getCommodityService(CommodityType.Goods);
    Map<String,String> extMap = new HashMap<String,String>();
    extMap.put("consigneeUserName", "谢飞机");
    commodityService_2.sendCommodity("10001","9820198721311","1023000020112221113", extMap);
    // 3. 第三方兑换卡(爱奇艺)
    ICommodity commodityService_3 = storeFactory.getCommodityService(CommodityType.Card);
    commodityService_3.sendCommodity("10001","AQY1xjkUodl8LO975GdfrYUio",null,null);
}
